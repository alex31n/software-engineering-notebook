# Hash Table

A **Hash Table** (or Hash Map) is a data structure that implements an associative array abstract data type, a structure that can map keys to values. It uses a **hash function** to compute an index (also called a hash code) into an array of buckets or slots, from which the desired value can be found.

Imagine you are working in a giant library. Instead of searching aisle by aisle for a specific book title, you give the title to a super-smart librarian (the hash function). The librarian instantly calculates a unique aisle and shelf number based on the letters in the title, and tells you exactly where to go. You go straight to that shelf and get your book! 

## How it Works

1.  **Key-Value Pair:** You have a `Key` (like a book title or student ID) and a `Value` (like the book contents or student name).
2.  **Hash Function:** You pass the `Key` into a Hash Function. The hash function processes the key and outputs an integer index.
3.  **Storage:** The `Value` is stored in the underlying array at the index generated by the hash function.
4.  **Retrieval:** To look up a value, you pass the `Key` through the same hash function, get the same index, and directly access the value in the array.

### Collisions
What happens if two different keys produce the exact same index from the hash function? This is called a **Collision**. Since the array size is limited, collisions are inevitable. 

There are two main ways to handle collisions:
*   **Chaining (Separate Chaining):** Each bucket in the array points to a linked list (or another data structure). If multiple keys hash to the same bucket, their key-value pairs are stored as nodes in that linked list.
*   **Open Addressing:** If a collision occurs, the hash table searches for the next available (empty) slot in the array and stores the key-value pair there. (Common methods include Linear Probing, Quadratic Probing, and Double Hashing).

## Complexity
*   **Time Complexity:**
    *   **Average Case:** $O(1)$ for Search, Insert, and Delete. The hash function instantly calculates the index, leading to constant time access.
    *   **Worst Case:** $O(n)$ for Search, Insert, and Delete. This happens if *every* key hashes to the exact same index (a massive collision), effectively turning the hash table into a single linked list (when using chaining). A good hash function and proper resizing minimize this risk.
*   **Space Complexity:** $O(n)$, where $n$ is the number of key-value pairs stored in the hash table.

## Implementation

To build a solid understanding, we will first look at a **Basic Implementation** that shows the core concept of hashing but *does not* handle collisions (meaning data could be overwritten). Then, we will look at an **Advanced Implementation** that properly handles collisions using **Separate Chaining**.

### Python

#### 1. Basic Implementation (No Collision Handling)
This version simply uses an array to store key-value pairs based on their hash. If a collision happens, the old value is lost!

```python
class BasicHashTable:
    def __init__(self, capacity=10):
        self.capacity = capacity
        # An array of empty slots
        self.buckets = [None] * self.capacity

    def _hash(self, key):
        # Calculate index using Python's built-in hash function
        return hash(key) % self.capacity

    def insert(self, key, value):
        index = self._hash(key)
        # DANGER: If two keys hash to the same index, the old data is overwritten!
        # This is a collision, and this basic version does not handle it.
        self.buckets[index] = [key, value]

    def get(self, key):
        index = self._hash(key)
        pair = self.buckets[index]
        if pair is not None and pair[0] == key:
            return pair[1]
        return None

# Example usage:
simple_table = BasicHashTable()
simple_table.insert("apple", 100)
simple_table.insert("banana", 200)

print(f"Apple price: {simple_table.get('apple')}")
```

#### 2. Advanced Implementation (Handling Collisions with Separate Chaining)
Here we use an array of Linked Lists. If multiple keys hash to the same index, we just add a new node to the list at that index.

```python
class Node:
    def __init__(self, key, value):
        self.key = key
        self.value = value
        self.next = None

class HashTable:
    def __init__(self, capacity=10):
        self.capacity = capacity
        # Create an array of empty buckets (Linked List heads)
        self.buckets = [None] * self.capacity
        self.size = 0

    def _hash(self, key):
        return hash(key) % self.capacity

    def insert(self, key, value):
        index = self._hash(key)
        head = self.buckets[index]

        # Check if key already exists, update value
        current = head
        while current:
            if current.key == key:
                current.value = value
                return
            current = current.next

        # Key doesn't exist, insert new node at the beginning of the list
        new_node = Node(key, value)
        new_node.next = head
        self.buckets[index] = new_node
        self.size += 1

    def get(self, key):
        index = self._hash(key)
        current = self.buckets[index]

        while current:
            if current.key == key:
                return current.value
            current = current.next
        
        return None # Key not found

    def remove(self, key):
        index = self._hash(key)
        current = self.buckets[index]
        previous = None

        while current:
            if current.key == key:
                if previous:
                    previous.next = current.next
                else:
                    self.buckets[index] = current.next
                self.size -= 1
                return True
            previous = current
            current = current.next
            
        return False # Key not found

# Example usage:
my_hash_table = HashTable()
my_hash_table.insert("apple", 100)
my_hash_table.insert("banana", 200)
my_hash_table.insert("orange", 300)

print(f"Apple price: {my_hash_table.get('apple')}")     # Output: 100
my_hash_table.remove("apple")
print(f"Apple price after removal: {my_hash_table.get('apple')}") # Output: None
```

### Java

#### 1. Basic Implementation (No Collision Handling)
A simple array-based approach.

```java
class SimpleEntry {
    String key;
    Integer value;

    public SimpleEntry(String key, Integer value) {
        this.key = key;
        this.value = value;
    }
}

public class BasicHashTable {
    private SimpleEntry[] buckets;
    private int capacity;

    public BasicHashTable(int capacity) {
        this.capacity = capacity;
        this.buckets = new SimpleEntry[capacity];
    }

    private int getHash(String key) {
        int index = key.hashCode() % capacity;
        return index < 0 ? index * -1 : index;
    }

    public void insert(String key, Integer value) {
        int index = getHash(key);
        // DANGER: If two keys hash to the same index, the old data is overwritten!
        // This is a collision, and this basic version does not handle it.
        buckets[index] = new SimpleEntry(key, value);
    }

    public Integer get(String key) {
        int index = getHash(key);
        SimpleEntry entry = buckets[index];
        if (entry != null && entry.key.equals(key)) {
            return entry.value;
        }
        return null;
    }

    public static void main(String[] args) {
        BasicHashTable simpleTable = new BasicHashTable(10);
        simpleTable.insert("apple", 100);
        simpleTable.insert("banana", 200);
        
        System.out.println("Apple price: " + simpleTable.get("apple"));
    }
}
```

#### 2. Advanced Implementation (Handling Collisions with Separate Chaining)
Here, each bucket contains the head of a linked list.

```java
import java.util.ArrayList;

class HashNode<K, V> {
    K key;
    V value;
    HashNode<K, V> next;

    public HashNode(K key, V value) {
        this.key = key;
        this.value = value;
    }
}

public class HashTable<K, V> {
    private ArrayList<HashNode<K, V>> bucketArray;
    private int numBuckets;
    private int size;

    public HashTable() {
        bucketArray = new ArrayList<>();
        numBuckets = 10;
        size = 0;

        // Initialize empty buckets
        for (int i = 0; i < numBuckets; i++) {
            bucketArray.add(null);
        }
    }

    private int getBucketIndex(K key) {
        int hashCode = key.hashCode();
        int index = hashCode % numBuckets;
        // hashCode could be negative, so ensure positive index
        return index < 0 ? index * -1 : index;
    }

    public void insert(K key, V value) {
        int bucketIndex = getBucketIndex(key);
        HashNode<K, V> head = bucketArray.get(bucketIndex);

        // Check if key is already present
        HashNode<K, V> current = head;
        while (current != null) {
            if (current.key.equals(key)) {
                current.value = value;
                return;
            }
            current = current.next;
        }

        // Insert key in the chain
        size++;
        head = bucketArray.get(bucketIndex);
        HashNode<K, V> newNode = new HashNode<>(key, value);
        newNode.next = head;
        bucketArray.set(bucketIndex, newNode);
    }

    public V get(K key) {
        int bucketIndex = getBucketIndex(key);
        HashNode<K, V> head = bucketArray.get(bucketIndex);

        // Search key in chain
        while (head != null) {
            if (head.key.equals(key)) {
                return head.value;
            }
            head = head.next;
        }
        return null; // Key not found
    }

    public V remove(K key) {
        int bucketIndex = getBucketIndex(key);
        HashNode<K, V> head = bucketArray.get(bucketIndex);
        HashNode<K, V> prev = null;

        while (head != null) {
            if (head.key.equals(key)) {
                break;
            }
            prev = head;
            head = head.next;
        }

        // If key was not there
        if (head == null) {
            return null;
        }

        size--;
        // Remove key
        if (prev != null) {
            prev.next = head.next;
        } else {
            bucketArray.set(bucketIndex, head.next);
        }

        return head.value;
    }

    public static void main(String[] args) {
        HashTable<String, Integer> map = new HashTable<>();
        map.insert("apple", 100);
        map.insert("banana", 200);
        map.insert("orange", 300);

        System.out.println("Apple price: " + map.get("apple"));   // Output: 100
        System.out.println("Mango price: " + map.get("mango"));   // Output: null

        map.remove("apple");
        System.out.println("Apple price after removal: " + map.get("apple")); // Output: null
    }
}
```

## Using Built-in Hash Tables

In real-world software engineering, you rarely build Hash Tables from scratch. Modern programming languages provide highly optimized built-in implementations.

### Python (Dictionaries)
In Python, Hash Tables are implemented as **Dictionaries** (`dict`).

```python
# Create a dictionary
student_grades = {
    "Alice": "A",
    "Bob": "B",
    "Charlie": "A-"
}

# Insert or update a value (O(1))
student_grades["David"] = "B+"  # Insert
student_grades["Bob"] = "A"     # Update

# Retrieve a value (O(1))
print(f"Alice's grade: {student_grades['Alice']}")

# Safely retrieve a value (returns None if key doesn't exist)
print(f"Eve's grade: {student_grades.get('Eve')}")

# Remove a key-value pair (O(1))
del student_grades["Charlie"]

# Check if a key exists (O(1))
if "David" in student_grades:
    print("David is in the class.")
```

### Java (HashMap)
In Java, the most common Hash Table implementation is the `HashMap` class from the `java.util` package.

```java
import java.util.HashMap;
import java.util.Map;

public class BuiltInHashTableExample {
    public static void main(String[] args) {
        // Create a HashMap
        Map<String, String> studentGrades = new HashMap<>();

        // Insert or update a value (O(1))
        studentGrades.put("Alice", "A");
        studentGrades.put("Bob", "B");
        studentGrades.put("Charlie", "A-");
        
        studentGrades.put("Bob", "A"); // Update

        // Retrieve a value (O(1))
        System.out.println("Alice's grade: " + studentGrades.get("Alice"));

        // Safely retrieve with a default value
        System.out.println("Eve's grade: " + studentGrades.getOrDefault("Eve", "Not Found"));

        // Remove a key-value pair (O(1))
        studentGrades.remove("Charlie");

        // Check if a key exists (O(1))
        if (studentGrades.containsKey("David")) {
            System.out.println("David is in the class.");
        }
    }
}
```

## Advantages & Disadvantages
**Advantages:**
- Extremely fast lookups, insertions, and deletions ($O(1)$ on average).
- Excellent for scenarios where you need to check for existence, keep track of frequencies, or map unique identifiers to data.

**Disadvantages:**
- Not ideal for maintaining an ordered set of data (elements are stored arbitrarily based on hash).
- If the hash function is poor or the table becomes too full, collisions increase, degrading performance to $O(n)$.
- Can require more memory than other data structures due to reserving array space that might remain empty to maintain a low load factor.
