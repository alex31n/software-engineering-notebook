# Design Patterns

This documentation contains notes on various software design patterns.

## 1. Creational Patterns
Creational design patterns provide various object creation mechanisms, which increase flexibility and reuse of existing code.

1. [Abstract Factory](creational/abstract-factory.md)
2. [Builder](creational/builder.md)
3. [Factory](creational/factory.md)
4. [Prototype](creational/prototype.md)
5. [Singleton](creational/singleton.md)

## 2. Structural Patterns
Structural design patterns explain how to assemble objects and classes into larger structures while keeping these structures flexible and efficient.

1. [Adapter](structural/adapter.md)
2. [Composite](structural/composite.md)
3. [Decorator](structural/decorator.md)
4. [Facade](structural/facade.md)
5. [Proxy](structural/proxy.md)

Behavioral design patterns are concerned with algorithms and the assignment of responsibilities between objects.

2. [Iterator](behavioral/iterator.md)

4. [Observer](behavioral/observer.md)
